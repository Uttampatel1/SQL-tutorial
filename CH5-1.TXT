SQL> REM EXECUTION PRECEDENCE OF SELECT : FROM -> WHERE -> GROUP BY -> HAVING -> ORDER BY -> OFFSET -> FETCH -> SELECT
SQL> 
SQL> 
SQL> REM FROM : TO READ ALL ROWS OF TABLE FROM HDD AND PUT IT INTO MEMORY/RAM
SQL> 
SQL> REM WHERE : TO REMOVE ALL UNWANTED ROWS FROM MEMORY, WHICH DOESN'T SATISFY 'WHERE' CONDITION
SQL> 
SQL> REM GROUPY : WILL GROUP ROWS ON THE BASES OF VALUES OF GROUP BY COLUMN(S)
SQL> 
SQL> REM GROUP BY : WILL GROUP ROWS ON THE BASES OF VALUES OF GROUP BY COLUMN(S) AND GENERATE SUMMARY ROW FOR EACH GROUP
SQL> 
SQL> REM HAVING : WILL REMOVE ALL UNUWANTED SUMMARY ROWS WHICH DOESN'T SATISFY 'HAVING' CONDITION
SQL> 
SQL> REM ORDER BY : TO SORT ROWS IN ASC/DESC ON SINGLE/MULTIPLE COLUMNS, BY KEEPING NULLS AT THE TOP/BOTTOM
SQL> 
SQL> REM OFFSET : TO FIX STARTING ROW OF OUR RESULT SET
SQL> 
SQL> REM FETCH : TO FIX NO. OF ROWS TO BE DISPLAYED FOR OUR RESULT SET
SQL> 
SQL> REM SELECT : TO ACTUALLY DISPLAY THE DATA/ROWS ON SCREEN
SQL> 
SQL>
